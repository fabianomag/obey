# OBEY

**Tagline:** Make AI obey.

## What this project is

OBEY is not a general "AI consultant" brand.
OBEY is not a note-taking memory app.
OBEY is not another prompt library.

OBEY is the product layer for a harder problem:

**making agents respect context, decisions, references, and approvals across real work.**

---

## Why OBEY exists

People already have access to ChatGPT, Claude, Codex, and other strong models.

The unsolved pain is no longer "how do I talk to an AI?"

The unsolved pain is:

- why does the agent forget what was decided?
- why does it drift from approved references?
- why does it regress into ideas we already rejected?
- why does it require so much supervision to stay coherent?
- why is there still no reliable execution flow after a good answer?

OBEY exists to solve that layer.

---

## Brand architecture

### OBEY

Public product brand.

This is what the market sees.

### COS

Internal engine and orchestration layer.

COS is not the main public brand anymore.
It remains important as internal architecture, learning loop, and founder operating system.
In practice, COS should be treated as an internal skill and workflow layer that operationalizes frontier coding agents like Claude Code and Codex.

### SGOPS

Research lab, editorial surface, and archive.

SGOPS documents:

- why agents fail
- what we learned from real projects
- how reference-grounded execution works
- why "memory" is not enough
- where the ecosystem is going

SGOPS becomes the lab/media surface of the ecosystem, not the main commercial wedge.

---

## Product shape

### OBEY OSS

Free surface.

Purpose:

- attract builders and agent users
- demonstrate the thesis
- improve agent behavior in a visible way
- drive adoption and discussion

### OBEY Runtime

Paid or monetizable surface.

Purpose:

- persist project state
- store decisions and approvals
- route tasks through execution contracts
- attach context/reference packs
- enable repeatable agent execution

### COS Skill Layer

Internal, founder-facing layer.

Purpose:

- operationalize Claude Code and Codex
- encode founder workflow rules
- compress sessions into reusable project state
- help Frank use agents with more continuity and less drift
- help keep current operating priorities moving proactively

Important:

this layer may create future public insights or reusable pieces,
but it does not need to be the main commercial artifact on day one.

Current emphasis:

the first high-value COS mode is not "do everything."
it is helping run the authority and distribution machine around OBEY and SGOPS.

---

## The real wedge

OBEY does not try to beat the frontier labs at general intelligence.

OBEY wins if it becomes the layer that they are missing:

**the obedience layer for agents.**

That means:

- Claude Code and Codex are engines
- COS is the internal operationalizer
- OBEY is the externalized product layer

---

## Internal does not mean useless

COS being internal is not a weakness.

It is the private operating layer that lets Frank do better work with Claude Code and Codex.

That private layer can create value in three different ways:

- service value: Frank uses COS internally to deliver better outcomes for real clients
- product value: OBEY externalizes the reusable parts of that method for other builders
- media value: SGOPS publishes the lessons, failures, and frameworks that emerge from real use

This means COS does not need to be sold directly on day one in order to matter.

It only needs to make the work measurably better and feed artifacts that can later become public.
