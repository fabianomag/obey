# COS Operating State

This folder is the first practical scaffold for COS.

The goal is not to create a full product here.

The goal is to give COS a stable internal operating surface so it can be:

- proactive
- stateful
- mode-based
- easier to trust

## Principle

Natural-language systems become more reliable when they operate over explicit state.

That state lives here.

## Files

- `CURRENT_PRIORITY.md` — what matters right now
- `POSITIONING.md` — approved language and core claims
- `CONTENT_QUEUE.md` — queued and approved content work
- `AUTHORITY_BACKLOG.md` — strategic proof and authority ideas
- `WEEKLY_CONTENT_PACK.md` — the current weekly packaging target

## Current use

For now, the dominant COS mode is:

`Marketing Operations`

This means COS should prioritize:

- turning work into proof
- keeping authority surfaces moving
- reducing publishing paralysis

## Rule

If a new request does not relate to the current priority, COS should not derail the week.

It should capture the idea and send it to backlog instead.
